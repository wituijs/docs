# witMount 应用挂载组件 %{#witMount}%

`witMount`是一个基于 WujieVue 微前端框架封装的应用挂载组件，用于初始化和管理微应用的加载配置，支持动态设置应用URL和预加载功能，实现主应用与微应用的无缝集成。

## 用法实例 %{#dome}%

```vue
<template>
  <div>
    <!-- 在应用入口处使用，自动初始化所有微应用配置 -->
    <witMount />
  </div>
</template>

<script lang="ts" setup>
// 组件无需额外配置，自动从全局应用列表读取配置信息
</script>
```

## API

| 属性名                 | 类型            | 默认值                     | 描述                                           |
| :--------------------- | :-------------- | :------------------------- | :--------------------------------------------- |
| 无                     | -               | -                          | 组件无需外部传入属性，自动从全局配置获取        |

## 插槽

| 插槽名                 | 描述                                           |
| :--------------------- | :--------------------------------------------- |
| 无                     | 组件不提供插槽                                 |

## 事件

| 事件名                 | 描述                                           | 参数                         |
| :--------------------- | :--------------------------------------------- | :--------------------------- |
| 无                     | 组件不触发额外事件                             | -                            |

:::tip
组件依赖以下配置和环境：
1. 全局应用列表 `apps`，包含微应用的名称、URL路径和预加载配置
2. WujieVue 微前端框架，提供 `setupApp` 和 `preloadApp` 核心方法
3. 开发环境下依赖全局变量 `window.g.wujieURL` 作为URL前缀
4. 生产环境下使用当前页面的协议和域名作为URL前缀

应用配置中的 `preload` 属性决定了应用是否会在组件挂载时进行预加载，以提升用户体验。
:::
