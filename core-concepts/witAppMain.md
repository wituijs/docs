# witAppMain 应用主内容组件 %{#witAppMain}%

`witAppMain` 是一个应用主内容区域组件，作为路由视图的容器，负责渲染应用的主要页面内容，并与路由状态管理集成，实现路由变化时的状态更新。

## 用法实例 %{#dome}%

```vue
<template>
  <witAppMain />
</template>

<script lang="ts" setup>
import witAppMain from '@/library/components/witAppMain'
</script>
```

## 功能特性

### 路由视图容器
- 作为应用的主要内容区域，渲染匹配的路由组件
- 使用 `wit-router-view` 组件展示页面内容
- 提供统一的内容区域布局结构

### 路由状态管理
- 集成路由状态管理模块，监听路由变化
- 自动更新当前激活的标签页状态
- 动态设置当前激活的菜单项目
- 确保路由与UI状态的同步

### 布局结构
- 采用标准的布局结构，包含 `wit-app-main` 和 `section` 容器
- 为应用内容提供统一的样式基础
- 支持与其他布局组件的无缝集成

:::tip
1. 组件通常作为应用布局的主要内容区域使用
2. 依赖路由系统和路由状态管理模块
3. 会自动监听路由变化并更新相关状态
4. 作为路由视图的容器，需与 Vue Router 配合使用
5. 支持与导航菜单、标签页等组件协同工作
:::
