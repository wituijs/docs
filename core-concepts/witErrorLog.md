# witErrorLog 错误日志组件 %{#witErrorLog}%

`witErrorLog`是一个用于捕获和展示应用异常错误的组件，通过错误日志状态管理，自动显示错误数量徽章，并提供错误详情查看和日志清空功能，帮助开发者及时发现和解决应用中的问题。

## 用法实例 %{#dome}%

```vue
<template>
  <div>
    <!-- 错误日志组件会自动显示在页面上 -->
    <witErrorLog />
  </div>
</template>

<script lang="ts" setup>
// 组件会自动捕获应用中的异常，无需额外配置
</script>
```

## API

| 属性名                 | 类型            | 默认值                     | 描述                                           |
| :--------------------- | :-------------- | :------------------------- | :--------------------------------------------- |
| 无                     | -               | -                          | 组件无需额外属性配置，自动工作                   |

## 事件

| 事件名                 | 描述                                           | 参数                         |
| :--------------------- | :--------------------------------------------- | :--------------------------- |
| 无                     | 组件内部处理所有交互，无需外部监听事件         | -                            |

:::tip
1. 组件依赖于 `errorLogStore` 状态管理，自动捕获应用中的异常错误。
2. 当有错误发生时，会在页面上显示带有错误数量的徽章图标。
3. 点击徽章图标会打开错误日志对话框，展示详细的错误信息。
4. 提供"暂不显示"按钮，可以清空当前的错误日志。
5. 组件使用了 `wit-dialog` 和 `wit-error-log-content` 组件来展示错误详情。
:::
